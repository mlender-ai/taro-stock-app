## P1 · 개인화된 FOMO 변화 피드백 알림 시스템
- **status**: backlog
- **milestone**: 리텐션 강화
- **okr**: DAU 증가, 알림 열람률 20% 달성
- **why**: 사용자들이 자신의 감정 변화 및 집단 데이터를 실시간으로 알림을 통해 확인한다면 앱과의 상호작용 빈도가 높아지고 리텐션이 강화될 것이다.
- **success**: 사용자에게 하루 단위 또는 주요 변화 발생 시 FOMO Index 및 개인 감정 변화 요약 푸시 알림이 정상적으로 발송된다.
- **scope**: 푸시 알림 발송 시스템 개발(API/스케줄러) 및 앱 알림 UI 추가
- **breakdown**: 기획: 알림 이벤트(예: 개인 감정 변화 >5포인트, 집단 FOMO Index 변화 >10포인트 등) 정의 / 백엔드: 알림 스케줄러 개발 및 클라이언트 피드 노출 API 제공 / 프론트·UX: 알림 설정 및 현황 관리 기능 UI 구현 / 품질: 알림 발송 정확성과 시뮬레이션 테스트

## P2 · 감정 중심의 게임화된 데일리 챌린지
- **status**: backlog
- **milestone**: 사용자 행동 유도 강화
- **okr**: 사용자 재방문율 10% 증가
- **why**: 사용자들이 매일 접속하여 감정과 FOMO Index 개선에 관심을 가지도록 게임화된 목표 설정 및 보상을 제공할 필요가 있음.
- **success**: 사용자가 매일의 감정 평가 후 개인의 포모 지수를 개선하는 미션을 수행하고 보상을 받을 수 있음.
- **scope**: 데일리 챌린지 로직 설계, 챌린지 완료 화면 및 포인트 시스템 추가, 챌린지 상태 저장을 위한 데이터베이스 테이블 추가
- **breakdown**: 기획: 데일리 챌린지 플로우 설계 및 난이도 밸런싱 / 백엔드: 챌린지 상태 데이터베이스 구축 및 상태 동기화 API 생성 / 프론트·UX: UI 설계 및 피드백 애니메이션 구현 / 품질: 플로우 유효성 검증 및 포인트 적립 정확성 확인

## P3 · FOMO 모의 실험 시뮬레이터
- **status**: backlog
- **milestone**: 교육 및 탐색 기능 강화
- **okr**: 사용자 생성 시뮬레이션 참여 건 수 500건 달성
- **why**: FOMO Index가 변할 때 추정되는 사용자 감정 반응을 예측하여 교육 및 놀이 요소를 추가할 기회가 있음.
- **success**: FOMO 모의 실험 기능이 출시되어 사용자가 가상의 시장 상황 변화를 기반으로 Index 및 감정 변화를 시뮬레이션 가능함.
- **scope**: 사용자 정의 변수 입력 UI 추가, 시뮬레이션 연산 백엔드 로직 설계 및 결과 시각화
- **breakdown**: 기획: 시뮬레이터 기능 정의 및 요구사항 정리 / 백엔드: 모의 연산 API 및 입력 변수 처리 로직 개발 / 프론트·UX: 사용자 입력 화면 및 결과 가시화 UI 설계 / 품질: 시나리오 타당성 테스트 및 시각화 검증

## P4 · 실시간 집단 감정 변화 트리거 피드
- **status**: backlog
- **milestone**: 사용자 피드 인터랙션 향상
- **okr**: 피드 조회 수 1000 건 달성
- **why**: 커뮤니티형 서비스로 사용자들이 실시간으로 감정 변화와 관련된 정보를 확인하면 앱 체류 시간이 길어질 것으로 예상됨.
- **success**: 실시간으로 감정 변화 트리거(예: 특정 감정 급증) 노출 및 해당 원인 확인 가능
- **scope**: 실시간 감정 변화 트리거 데이터 수집/분석 시스템, 트리거 피드 UI, 관련 게시물 연동
- **breakdown**: 기획: 감정 변화 트리거 발생 조건 및 표시 방법 정의 / 백엔드: 실시간 데이터 수집 및 트리거 알림 API 개발 / 프론트·UX: 트리거 피드 화면 UI/UX 디자인 / 품질: 데이터 정확성 검증 및 트리거 조건 테스트

## 진행 원칙
- **활성은 동시 1개.** 새 프로젝트 선택 시 이전 active 는 backlog 로 강등(완료는 명시적 done 처리).
- 활성 프로젝트가 있으면 직군은 그 프로젝트 이슈만 쪼개고 격파한다. 산발적 일일 제안 금지.
- 활성 프로젝트가 없으면 daily 리포트는 백로그를 surfacing 하고 오너의 선택을 기다린다.
- 프로젝트 완료 → retro → 다음 후보 surfacing.
