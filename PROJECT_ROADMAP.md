# 🗺️ FOMO Club 프로젝트 로드맵 (Project Backlog)

| | |
|---|---|
| **위상** | 톱다운 프로젝트 백로그의 **단일 진실(source of truth)**. 에이전트는 매일 잔 아이디어를 던지지 않고, 여기 **활성 프로젝트** 하나를 향해 움직인다. |
| **소유** | CEO(인간 오너)가 활성 프로젝트를 선택한다. Product Lead 에이전트가 백로그 후보를 *제안*(governance PR)한다. |
| **선택 방법** | Slack 봇에 "P1 프로젝트 시작" → `[[ACTION:select_project]] {"id":"P1"}`. 선택 시 `project-kickoff` 워크플로가 status=active 로 갱신하고 4축이 프로젝트를 이슈로 분해한다. |
| **상태값** | `active`(지금 격파 중, 동시 1개) · `backlog`(대기) · `done`(완료) |

> 프로젝트 = 마일스톤(`docs/IDENTITY_AND_MILESTONES.md` §5) 단위. "기능 완성"이 아니라 "사랑스러움 완성" 기준.
> 파싱 규칙: `## <id> · <title>` 헤딩 + 그 아래 `- **key**: value` 라인. (scripts/project-roadmap.ts)

## P1 · 단 하나의 사랑스러운 순간
- **status**: backlog
- **milestone**: M1
- **okr**: O1, O3
- **why**: 물렸을 때 포모를 열고 "너만 그런 게 아니야"라는 안도를 받는 단 하나의 순간. 이게 없으면 제품의 심장이 없다.
- **success**: 창업자 본인이 "내가 물렸던 그날 밤 진짜 열고 싶다"고 느낀다. 홈 1화면에 포모 표정(시장의 포모)→감정 선택→나의 포모 반응→담담한 한마디가 love mark 수준으로 완성.
- **scope**: 포모 마스코트(담담한 표정·Index 연동) · 2단계 감정 변화(시장의 포모→나의 포모) · 담담한 솔직함 한마디(가짜긍정❌·거침❌) · 전환 애니메이션 love mark · 시장 크립토 1개로 시작

## P2 · 매일 돌아올 이유
- **status**: backlog
- **milestone**: M2
- **okr**: O3
- **why**: 단 하나의 순간을 매일의 습관으로. 변동성 없는 날에도 포모를 열 이유.
- **success**: 잔잔한 날에도 포모를 여는 사람이 생긴다.
- **scope**: 감정 캘린더(한 달 픽셀 그리드) · 빈 칸 채우기+스트릭 · 시장 FOMO Index 흐름 옅게 겹침 · 잔잔한 날=치유의 날

## P3 · 혼자가 아님의 체감 (정직한 숫자)
- **status**: backlog
- **milestone**: M3
- **okr**: O2
- **why**: 집단 감정의 위로. 사용자 수가 적어도 "혼자가 아니다"가 체감되어야.
- **success**: 소수 사용자에서도 "혼자가 아니다"가 체감됨. 정직한 숫자 원칙 0건 위반.
- **scope**: "오늘 N명이 같은 감정"(실제 집계·무가입 웹) · 고래 트래킹 카드(콜드스타트 해결) · 카드 문법 1개 통일

## P4 · 불안 해소 여정의 완성
- **status**: backlog
- **milestone**: M4
- **okr**: O1, O3
- **why**: 물릴 때마다 다른 앱 전전하지 않고 여기서 사이클이 끝나게.
- **success**: 물렸을 때 여기서 끝남(이탈 없이 회복).
- **scope**: 감정 동반 피드(포모가 여정 안내자) · 회복적 경쟁 가드레일(무모함❌·의연함⭕) · 위로는 타인의 목소리로 · 구조화 한마디(docs/M4_EXECUTION_PLAN.md)

## P5 · 깊이의 확장 (검증 후)
- **status**: backlog
- **milestone**: M5
- **okr**: O4
- **why**: M1~M4가 사랑받는 게 확인된 후에만. 검증 없이 당기면 "제품에 대한 FOMO".
- **success**: M1~M4 검증 완료가 전제. (그 전엔 선택 금지)
- **scope**: 주식 시장 추가(같은 카드 틀) · 타로 엔진=심층 해석 레이어 흡수 · 사주팔자 투자체질 · 포모 육성/시즌

## 진행 원칙
- **활성은 동시 1개.** 새 프로젝트 선택 시 이전 active 는 backlog 로 강등(완료는 명시적 done 처리).
- 활성 프로젝트가 있으면 에이전트는 그 프로젝트 이슈만 쪼개고 격파한다. 산발적 일일 제안 금지.
- 활성 프로젝트가 없으면 daily 리포트는 백로그를 surfacing 하고 오너의 선택을 기다린다.
- 프로젝트 완료 → retro → 다음 후보 surfacing.
