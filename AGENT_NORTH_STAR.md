## ⛔ 절대 제안 금지 카테고리 (전 에이전트 자동 폐기 — 정렬도 무관)

North Star 정렬도가 5점이어도 이 카테고리에 해당하면 즉시 폐기:

- **가짜/임의 숫자**: 실제 집계 없는 수치 표기, 데이터 소스 없는 FOMO Index 값, 부풀린 사용자/참여 수 (FOMO_CLUB "정직한 숫자 원칙" 위반)
- **투자 조언·예측**: "오를 것이다/사라", 종목 추천, 매수·매도·목표가·매매 신호 (FOMO Index는 금융 지표가 아닌 감정 체감 지표 — 면책 위반)
- **장식 폴리싱**: BGM·효과음, 글로우·파티클·반짝임, 의미 없는 장식 애니메이션, 햅틱.
  → **단 예외**: 마스코트 '포모'의 표정 전환(FOMO Index 5구간↔표정)은 장식이 아니라 **핵심 경험**이다. 표정·2단계 감정 변화(시장의 포모→나의 포모) 관련은 금지 대상 아님.
- **커뮤니티/감정 피드**: MVP 범위 밖(FOMO_CLUB "커뮤니티 초기 미포함, Phase 2"). 집단 감정 수치로 대체.

→ "몰입감"·"바이럴" 같은 추상어로 위를 포장하는 것도 동일하게 폐기.

## 🎯 이번 주 집중 영역

1. **FOMO Index 신뢰성**
   - 4 Heat(Market 30 / Community 30 / Emotion 30 / Whale 10) 산출 파이프라인 (`@fomo/core`)
   - 데이터 소스 미비 시 안전한 기본값 + 폴백 — 빈 값/에러 절대 노출 금지
   - 5구간 ↔ 마스코트 포모 5표정 매핑 일관성 (경계값 변경 시 표정 동시 점검)

2. **홈 MVP 핵심 경험**
   - FOMO Index + 오늘의 감정 투표 + 집단 통계 + Market Pulse 배너 (한 화면)
   - 마스코트 포모의 표정 = Index, 2단계 감정 변화(시장의 포모 → 나의 포모)
   - 무가입 웹 방문 가능 (익명 세션), 정보 위계는 명확히·화면 밀도는 비운다

3. **정직한 숫자**
   - 가짜/임의 수치 0건. 작아도, 0이어도 실제 집계값만 표기.
   - FOMO Index 등 모든 수치는 실제 집계/시장 데이터 기반.

## 📐 프로젝트 사실 규약 (에이전트 필독 — 잘못된 가정 금지)

다음은 **검증된 사실**입니다. 이와 모순되는 제안은 잘못된 가설로 간주되어 거부됩니다.

### 인프라/스택
- 모노레포: `apps/fomo-club`(Expo, NativeWind)·`apps/fomo-web`(Next)·`packages/fomo-core`. **타로 엔진(`tarot-core`)·`tarot-mobile`은 보존 중(리네이밍 연기) — 삭제·재제안 금지.**
- `@fomo/core`: FOMO Index 타입 + 5구간 ↔ 포모 표정 매핑 (구간 경계 변경 시 MASCOT 매핑 동시 점검).
- DB 마이그레이션은 `prisma migrate`가 아닌 **`prisma db push`**로 운영.
- React Native는 **Expo Go 환경** — `react-native-reanimated` 사용 불가. `Animated` API만 허용.

### 이미 구현/정의된 것 (재제안 금지)
- FOMO Index 5구간 ↔ 포모 표정 매핑 (`@fomo/core`)
- `fomo-club` / `fomo-web` 앱 셸, FomoFace 컴포넌트
- 정직한 숫자 원칙, MVP 3화면 정의 (docs/FOMO_CLUB)
- (인프라 보존) JWT HttpOnly 쿠키, IAP 서버 검증, HMAC nonce 광고 검증, API 캐싱 — 타로 전용 기능은 "보존 중(리네이밍 연기)"

### 인지하고 있지만 이번 주는 보류
- AdMob 프로덕션 ID (인지됨 — 우선순위 아님)
- 트래킹 도구 부재 (현재 console.log)
- 타로 엔진 ↔ fomo-core 연계 (감정 집계 → 해석 재료, MVP 이후)

## ✍️ 제안 작성 절대 규칙 (모든 에이전트 공통)

### 표현
- ❌ **금지 어휘**: "가능성이 있음", "우려됩니다", "수도 있습니다", "보입니다", "~할 것 같습니다"
- ✅ **요구 어휘**: 단정형 + 근거. 근거가 없으면 제안하지 마세요.

### 근거
- 모든 "현재 문제" 진술은 **파일 경로 + 줄 번호** 또는 **메트릭 수치** 또는 **사용자 피드백 인용** 중 하나를 반드시 포함.
- 추측만으로 제안 금지. 근거 없으면 "조사 필요 — 다음 주"로 미루세요.

### 액션 단위
- 제안 1개는 **1주일 내 단일 PR로 완료 가능한 범위**.
- "리팩토링", "전반적 개선" 같은 추상 표현 금지. 함수명/파일 경로/구체 구현 명시.
  (예시: "카드 서사 강화" 류가 아니라 "감정 시각화·포모 표정 전환·집단 통계 표시" 류 FOMO 액션)

### North Star 정렬
- 제안 마지막에 반드시 **"이 제안이 이번 주 테마와 어떻게 연결되는가"** 1문장 + `Target-Metric`(registry id).
- 연결 안 되면 "다음 주 후보"로 표시하고 본 제안에서 제외.
