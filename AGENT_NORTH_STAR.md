## ⛔ 절대 제안 금지 카테고리 (전 에이전트 자동 폐기 — 정렬도 무관)

사용자(CEO)가 직접 지시한 자동 킬리스트. North Star 정렬도가 5점이어도 이 카테고리에 해당하면 즉시 폐기:

- **사운드**: 종소리, BGM, 효과음, 카드 뽑기 사운드, 결과 화면 사운드, "신비로운 사운드", 카운트다운 효과음
- **시각 효과**: 글로우, 파티클, 반짝임, 빛 효과, 발광 효과, 키 비주얼, 싱크로 효과
- **장식 애니메이션**: 결과 화면 등장 애니메이션, 텍스트 순차 페이드인, 카드 슬라이드/슬라이더 인터랙션, 카운트다운(3·2·1), 버튼 펄스/진동/팽창/수축, 미니 스토리 페이드 효과
- **햅틱 피드백**: Vibration, Haptics, 진동 패턴

→ "몰입감", "의식 경험", "신비로움" 같은 추상어로 위 카테고리를 포장해서 제안하는 것도 동일하게 폐기. **카드 뽑기 7장 부채꼴 동작·카드 뒤집기처럼 이미 구현된 핵심 인터랙션은 재제안조차 금지.**

## 🧪 핵심 가설

> 토스증권 수준의 정보 위계로 종목 데이터를 제공하고, 타로 해석 콘텐츠의 깊이를 끌어올리고, 백엔드 API가 그것을 정합성 있게 받쳐주면, D7 리텐션과 크레딧 재구매율이 오른다.
> 사운드/이펙트 같은 감각 폴리싱으로는 절대 그 지표를 못 움직인다 — 콘텐츠와 정보 구조가 본질이다.

## 🚫 이번 주 손대지 않을 것 (Out of Scope)

다음 영역의 제안은 자동 킬리스트로 간주합니다. 에이전트는 이 영역에 대한 제안을 만들지 마세요.

- **사운드/햅틱/장식적 시각 효과 추가** (2026-05-26 CEO 직접 지시) — 사운드 이펙트, BGM, 햅틱 피드백, 글로우/파티클/반짝임, 결과 화면 장식 애니메이션, 버튼 펄스·진동·빛 효과, 싱크로 효과. 카드 뽑기/뒤집기 같은 이미 구현된 핵심 인터랙션 동작 자체는 예외.
- **푸시 알림 관련 일체** (2026-05-27 CEO 직접 지시) — 푸시 카피·시간·빈도·페어링·A/B 모두 보류. 이번 단계는 콘텐츠 품질·종목 데이터 정확성·종목 상세 UX 강화가 먼저. 푸시는 그 기반 안정 이후 단계.
- 사주팔자 추가 (보류)
- 카카오톡 공유 기능 (보류)
- 어드민 UI 개선 (보류)
- 새 결제 상품 / IAP 변경 (안정 우선)
- 신규 화면 추가 (기존 화면 강화 우선)
- 새 인프라 도입 (Redis 등 — 비용 정당화 안 됨)

2. **콘텐츠 품질**
   - 타로 해석 텍스트의 깊이·구체성·재방문 동기
   - 카드 서사·문구 품질
   - 종목 데이터 표시 품질 (시세·차트·재무·뉴스 — 토스증권 수준)

3. **증권 UX 멘탈모델 = 토스증권 (tossinvest.com)**
   - 종목 상세·차트·재무·뉴스·공시 영역의 정보 위계·레이아웃·데이터 표기는 토스증권을 1차 레퍼런스로 한다

   - 토스와 동떨어진 금융 UX 제안은 근거 필요

## 📐 프로젝트 사실 규약 (에이전트 필독 — 잘못된 가정 금지)

다음은 **검증된 사실**입니다. 이와 모순되는 제안은 잘못된 가설로 간주되어 거부됩니다.

### 인프라/스택
- DB 마이그레이션은 **`prisma migrate`가 아닌 `prisma db push`**로 운영한다. → "마이그레이션 미배포" 같은 제안은 잘못된 가설.
- React Native는 **Expo Go 환경** — `react-native-reanimated` 사용 불가. `Animated` API만 허용.
- packages/tarot-core 의 현재 prompt 버전: `interpret-v1.1.0.ts`.

### 이미 구현된 것 (재제안 금지)
- JWT HttpOnly 쿠키 (7일 만료)
- IAP 서버 검증 (RevenueCat 연동)
- HMAC nonce 광고 검증
- API 응답 캐싱 (4-tier — market 5분, interpret 4-tier)
- 카드 뽑기 7장 부채꼴 선택 UI
- 7일 스트릭 보상 시스템
- 공유 보상 (1일 1크레딧)
- 앱스토어 리뷰 요청 프롬프트

### 인지하고 있지만 이번 주는 보류
- AdMob 프로덕션 ID (인지됨 — 이번 주 우선순위 아님)
- E2E 테스트 부재 (의식 경험 완성 후 처리)
- 트래킹 도구 부재 (현재 console.log — 다음 주)

## ✍️ 제안 작성 절대 규칙 (모든 에이전트 공통)

### 표현
- ❌ **금지 어휘**: "가능성이 있음", "우려됩니다", "수도 있습니다", "보입니다", "~할 것 같습니다"
- ✅ **요구 어휘**: 단정형 + 근거. 근거가 없으면 제안하지 마세요.

### 근거
- 모든 "현재 문제" 진술은 **파일 경로 + 줄 번호** 또는 **메트릭 수치** 또는 **사용자 피드백 인용** 중 하나를 반드시 포함해야 합니다.
- 추측만으로 제안 금지. 근거 없으면 "조사 필요 — 다음 주" 로 미루세요.

### 액션 단위
- 제안 1개는 **1주일 내 단일 PR로 완료 가능한 범위**여야 합니다.
- "리팩토링", "전반적 개선" 같은 추상 표현 금지. 함수명/파일 경로/구체 구현 명시.

### North Star 정렬
- 제안 마지막에 반드시 **"이 제안이 이번 주 테마와 어떻게 연결되는가"** 1문장 포함.
- 연결 안 되면 "다음 주 후보"로 표시하고 본 제안에서 제외.
