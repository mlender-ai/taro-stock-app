## 1. 한 줄 정의

사용자의 생년월일시(사주팔자)를 기반으로 **투자 체질 프로필**을 생성하고, 모든 타로 해석에 개인화 레이어를 추가하는 기능.

---

## 2. 왜 이 피처인가

### 현재 제품의 한계

타로 해석은 "종목 × 카드"의 조합이다. 같은 종목에 같은 카드를 뽑으면 누구에게나 비슷한 해석이 나온다. 개인화가 약하다.

### 사주가 해결하는 것

사주팔자는 "나라는 사람"을 분석한다. 이것이 타로 해석에 결합되면, 같은 카드를 뽑아도 **사용자마다 해석이 달라진다**. "나만의 해석"이 되는 순간, 대체 불가능한 제품이 된다.

### 한국 시장 적합성

사주, 궁합, 일진은 한국인에게 이미 친숙한 문화 프레임이다. 별도의 교육 비용 없이 "투자 사주"라는 컨셉이 즉시 이해된다. 20~40대 타겟 사용자의 운세 콘텐츠 소비 습관과 정확히 맞물린다.

## 3. 핵심 컨셉

### 3.1 투자 체질 프로필

생년월일시 → 사주팔자 계산 → 오행(五行) 분석 → 투자 체질 분류

**오행 × 투자 성향 매핑**:

| 오행 | 체질명 | 투자 성향 | 강점 | 약점 | 궁합 섹터 |
|---|---|---|---|---|---|
| 木 | 성장형 | 새싹을 키우는 장기 투자자 | 비전, 성장주 발굴, 인내 | 손절 어려움, 미련 | 테크, 헬스케어, 신재생 |
| 火 | 직관형 | 불꽃같은 모멘텀 투자자 | 빠른 판단, 트렌드 감지 | 조급, 과매매 | 에너지, 바이오, 엔터 |
| 土 | 안정형 | 대지같은 가치 투자자 | 꾸준함, 배당 중시, 안정 | 기회 놓침, 보수적 | 부동산, 인프라, 유틸 |
| 金 | 분석형 | 칼날같은 퀀트 투자자 | 논리, 리스크 관리, 데이터 | 결정 장애, 과분석 | 금융, 반도체, 소재 |
| 水 | 유연형 | 물처럼 흐르는 분산 투자자 | 적응력, 다양한 포트폴리오 | 줏대 부족, 분산 과다 | 물류, 해운, 소비재 |

### 3.2 사주 × 타로 크로스 해석

기존 타로 해석에 사주 프로필이 레이어로 추가되는 구조.

```
[기존] 종목 데이터 + 카드 메타 → AI 해석
[사주 결합] 종목 데이터 + 카드 메타 + 사용자 투자체질 → AI 맞춤 해석
```

동일 카드라도 사용자 체질에 따라 다른 해석:

- **火 체질 + 태양 카드**: "당신의 직관이 시장의 열기와 공명하고 있습니다. 하지만 火가 과하면 타오르다 꺼질 수 있어요. 오늘만큼은 직감보다 한 박자 늦추는 지혜가 필요합니다."
- **水 체질 + 태양 카드**: "시장의 뜨거운 기운이 당신의 차분한 물 기운과 만나 균형 잡힌 판단을 이끌어낼 수 있는 날입니다. 흐름을 따르되 깊이를 잃지 마세요."

### 3.3 일진(日辰) × 데일리 카드

매일의 천간·지지 조합이 사용자 사주와 상호작용하여 "오늘의 투자 기운"을 생성.

```
오늘의 일진: 丙午 (병오) — 火의 날

火 체질에게: "같은 기운이 겹치는 날. 에너지는 넘치지만 과열 주의."
水 체질에게: "물과 불이 부딪히는 날. 시장이 요동칠수록 고요히."
```

기존 데일리 시장 운세에 일진 해석이 자연스럽게 결합됨.

### 3.4 종목 궁합

사용자 오행과 종목/섹터 오행 간의 상생·상극 관계를 분석.

```
오행 상생 (궁합 좋음): 木→火→土→金→水→木
오행 상극 (궁합 주의): 木⊗土, 火⊗金, 土⊗水, 金⊗木, 水⊗火
```

종목의 섹터를 오행에 매핑하고, 사용자 주(主) 오행과의 관계로 궁합 점수 산출.

### 3.5 연간/분기 투자 운세 리포트

사주 기반 연간 투자 운세를 생성. 월별로 유리한/주의할 시기를 제시.

## 4. 사용자 플로우

```
[온보딩 또는 마이페이지에서]
생년월일 입력 (필수) + 태어난 시간 입력 (선택)
          ↓
사주 계산 → 투자 체질 프로필 생성
          ↓
투자 체질 카드 발급 (공유 가능)
          ↓
[이후 모든 타로 해석에 사주 프로필 자동 반영]
```

### 진입 시나리오

- **시나리오 A**: 온보딩 마지막 단계에서 선택적 입력 ("나만의 투자 체질을 알아보세요")
- **시나리오 B**: 마이페이지에서 별도 진입 ("투자 사주 분석")
- **시나리오 C**: 첫 해석 결과 화면에서 유도 ("사주를 입력하면 더 깊은 해석을 받을 수 있어요")

## 6. 기술 구현 계획 (구현 시점에 참조)

### 6.1 패키지 구조

```
packages/tarot-core/
  saju/                        ← 사주 전용 모듈 (Phase 6+에서 생성)
    ├── calculator.ts          ← 생년월일시 → 사주 팔자 계산
    ├── profile.ts             ← 사주 → 투자 체질 프로필 변환
    ├── daily-fortune.ts       ← 일진 계산 + 사주 상호작용
    ├── compatibility.ts       ← 사주 × 종목/섹터 오행 궁합
    ├── yearly-fortune.ts      ← 연간/분기 운세
    ├── constants.ts           ← 천간·지지·오행 상수 테이블
    └── types.ts               ← SajuProfile, Element, Pillar 타입
```

### 6.2 DB 스키마 (구현 시점에 추가)

```
SajuProfile {
  id            String   @id @default(cuid())
  userId        String   @unique
  birthYear     Int
  birthMonth    Int
  birthDay      Int
  birthHour     Int?              ← 모르면 null (시주 생략)
  mainElement   String            ← "wood" | "fire" | "earth" | "metal" | "water"
  investorType  String            ← "growth" | "intuitive" | "stable" | "analytical" | "flexible"
  pillars       Json              ← 4주 상세 (년주·월주·일주·시주)
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  
  user          User     @relation(fields: [userId], references: [id])
}
```

### 6.3 API 엔드포인트 (구현 시점에 추가)

```
POST /api/tarot/saju/profile     ← 생년월일시 → 프로필 생성
GET  /api/tarot/saju/profile     ← 내 프로필 조회
GET  /api/tarot/saju/daily       ← 오늘의 일진 + 내 사주 해석
GET  /api/tarot/saju/compatibility?ticker=005930  ← 종목 궁합
GET  /api/tarot/saju/yearly      ← 연간 운세
```

### 6.4 프롬프트 엔진 변경 (구현 시점에 추가)

```
[현재 USER 프롬프트]
{ marketData, card, spread }

[사주 추가 후 USER 프롬프트]
{ marketData, card, spread, sajuProfile: { mainElement, investorType, strengths, weaknesses } }
```

프롬프트 빌더 함수에 sajuProfile 파라미터를 **optional**로 추가하기만 하면 됨.

## 7. 현재 설계 시 고려사항

> **이 섹션이 가장 중요하다.** 모든 에이전트는 현재 코드를 작성할 때 아래 사항을 반영한다.

### 7.1 User 모델

User 모델에 사주 관련 필드를 지금 추가하지 않는다. 대신:
- User ↔ SajuProfile 은 나중에 1:1 relation으로 연결될 것이므로, User 모델에 사주 필드를 직접 넣는 설계를 하지 않는다.
- User 테이블에 birthDate 같은 필드를 선행 추가하지 않는다. SajuProfile 테이블에서 관리할 것이다.

### 7.2 프롬프트 엔진

해석 생성 프롬프트를 설계할 때:
- `buildPrompt()` 함수의 파라미터를 **확장 가능한 구조**로 만든다 (e.g. 옵셔널 `context` 객체).
- 사주 프로필이 나중에 context에 추가될 수 있음을 감안한다.
- 프롬프트 템플릿을 하드코딩하지 않고, 조합 가능한 섹션 블록으로 설계한다.

```typescript
// 좋은 예: 확장 가능
interface InterpretationInput {
  marketData: MarketSnapshot;
  card: TarotCard;
  spread: SpreadType;
  context?: Record<string, unknown>;  // ← 나중에 sajuProfile 등 추가
}

// 나쁜 예: 확장 불가
function buildPrompt(marketData, card, spread) { ... }
```

### 7.3 해석 결과 저장

Draw 모델의 `interpretation` 필드(Json):
- 현재는 `{ summary, detail, keywords }` 구조.
- 나중에 `{ summary, detail, keywords, sajuLayer?: { ... } }` 가 추가될 수 있으므로, interpretation을 고정 스키마로 타입 좁히지 않는다.
- Json 필드를 유지하되, TypeScript에서는 확장 가능한 인터페이스로 정의한다.

```typescript
// 좋은 예
interface Interpretation {
  summary: string;
  detail: string;
  keywords: string[];
  [key: string]: unknown;  // ← 확장 허용
}

// 나쁜 예
type Interpretation = { summary: string; detail: string; keywords: string[] };  // 닫힘
```

### 7.4 데일리 카드

데일리 카드 콘텐츠 구조를 설계할 때:
- 현재는 전체 시장 1장 해석이지만, 나중에 사용자별 일진 해석이 추가됨.
- 데일리 콘텐츠를 "공통 해석 + 개인 레이어" 구조로 분리 가능하게 설계한다.

### 7.5 종목/섹터 데이터

종목/섹터를 다룰 때:
- 섹터에 오행(Element)을 나중에 매핑할 수 있도록, 섹터 분류 체계를 명확히 유지한다.
- 종목 데이터에 `sector` 필드가 항상 포함되도록 한다.

### 7.6 UI 컴포넌트

마이페이지를 설계할 때:
- "투자 사주" 섹션이 나중에 추가될 공간을 고려한다.
- 프로필 카드 영역에 체질 뱃지/아이콘이 들어갈 수 있음을 감안한다.

해석 결과 화면을 설계할 때:
- 해석 텍스트 영역이 "기본 해석 + 사주 해석" 2블록 구조로 확장 가능해야 한다.
- 현재는 1블록만 표시하되, 나중에 사주 블록이 아래에 추가됨.

### 7.7 콘텐츠 기획

content-planner 에이전트는:
- 카드별 해석 문구 작성 시, 오행별 변형이 나중에 추가될 것을 인지한다.
- 하지만 지금은 오행별 변형을 작성하지 않는다. 범용 해석만 작성한다.
- 데일리 카드 콘텐츠에 "개인 일진" 슬롯이 추가될 것을 인지하되, 지금은 공통 해석만 작성한다.

## 8. 경쟁 우위 요약

```
1. 대체 불가능한 개인화 — 생년월일은 바뀌지 않으므로 "나만의 해석"
2. 한국 문화 적합성 — 사주/궁합/일진은 이미 친숙한 프레임
3. 리텐션 극대화 — "오늘의 일진"이 매일 앱을 여는 이유
4. 바이럴 소재 — "내 투자 체질은 火" 공유 욕구 자극
5. 크레딧 소비처 확대 — 궁합, 연간 리포트 등 고가 콘텐츠
6. 경쟁자 진입장벽 — 타로 + 사주 + 시장 데이터 3중 결합
```
