# FOMO Club 측정 정의

> 범위: 트랙 2 측정 인프라. 제품·화면·카피 로직을 변경하지 않는다.

## 현재 측정 가능한 이벤트

`TasteSignal`은 이미 운영 DB에 적재된다.

| 제품 행동 | 저장 신호 |
|---|---|
| 관심 스와이프 | `MORE` |
| 덜 관심 스와이프 | `LESS` |
| 테마·종목 상세 열람 | `VIEW_DEPTH` |
| 연관 종목 열람 | `TAP_RELATED` |

`npm run metrics:fomo`는 외부 분석 서비스 없이 위 데이터를 읽어 다음을 출력한다.

- 참여 사용자 기준 D1/D7 리텐션
- 30일 활성 참여자·참여 세션
- 세션당 스와이프 수
- 상세 열람이 발생한 세션 비율

세션은 같은 사용자 행동 간격이 30분을 초과하면 새 세션으로 정의한다. 날짜는 KST 기준이다.

## 아직 측정하지 않는 이벤트

- 순수 방문·재방문
- 카드 노출·완독
- 추천 위치와 개인화 모델 버전
- FOMO 가입 화면 노출·가입 완료·전환율

이 이벤트들은 화면 파일 수정 없이 일부 수집할 수 있지만, 별도 이벤트 테이블과 개인정보 보존 정책이 필요하다. `ProductEvent` DDL과 보존 기간 승인을 받은 뒤 추가한다. 승인 전에는 `TasteSignal`이나 감사 로그에 방문 데이터를 섞어 저장하지 않는다.

공용 `User` 테이블에는 레거시 제품 계정도 섞여 있으므로 `User.createdAt`을 FOMO 가입 수로 간주하지 않는다.

## 개인정보 원칙

- 이메일·IP·User-Agent를 제품 이벤트 속성에 저장하지 않는다.
- 익명 식별자는 기존 임의 `sessionId`만 사용한다.
- 원시 이벤트 보존 기간과 삭제 정책을 DDL 승인 전에 확정한다.
- 외부 분석 SaaS는 비용·국외 이전·쿠키 정책 승인을 받은 뒤 도입한다.
