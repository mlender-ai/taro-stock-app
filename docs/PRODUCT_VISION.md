## 1. 한마디 정의

### 내부 좌표 (팀·문서·에이전트용)
**매일 시장이 뭐에 들떴는지 쉽게 보고, 그래서 흔들리는 내가 따라가도 될 상태인지 판단하게 해주는 앱.**

### 외부 한 줄 (SNS·랜딩·피칭용)
**"종목을 분석하는 앱은 많아요. 포모클럽은 그 종목에 흔들리는 당신을 분석해요."**

### 정의의 두 절이 하는 일
- 앞 절("시장이 뭐에 들떴는지 쉽게") = **매일 오는 이유**(훅). 정보는 입장권.
- 뒤 절("흔들리는 내가 따라가도 될 상태인지") = **아무도 안 하는 차별점**(해자·BM). 진짜 파는 것.
- 핵심 전환: "정보를 캐치"로 정의하면 레드오션(토스·증권앱)에 빨려든다. 정보로는 못 이긴다. 차별점은 "종목 판단"이 아니라 **"흔들리는 나 자신의 판단"**에 있다.

## 2. 포지셔닝 (레드오션 안에서 안 겹치는 자리)

두 축: 정보 전달 ↔ 감정 다룸 / 종목을 판단 ↔ 흔들리는 나를 판단.

| | 종목을 판단 | 흔들리는 나를 판단 |
|---|---|---|
| **정보 전달** | 토스·증권앱 (빠르고 정확한 정보) | (빈칸) |
| **감정 다룸** | 유튜브 시황 (불안 증폭) / AI 분석 앱 (사라·마라) | **포모클럽 (흔들리는 나를 진정시켜 판단)** |

- 포모클럽만 "감정 × 자기 판단" 사분면에 단독 위치.
- **이 자리가 비어 있는 건 아무도 못 해서가 아니라, 큰 회사가 못 들어와서다.** 토스·증권사는 거래로 돈을 벌어서 "안 사도 괜찮아"를 절대 못 판다(자기모순). 거래 수수료가 없는 사이드 프로젝트가 오히려 들어갈 수 있는 자리.
- 엔진(뉴스·시장 데이터 수집)은 경쟁사와 비슷하다. 그 데이터를 **종목 평가가 아니라 내 감정 진단**에 쓰는 것이 포지션을 가르는 유일한 축.

## 3. 핵심 가설 (재정의)

> ~~매일 새로운 인사이트 + 진정 → 재방문~~ (약함: 신선함은 흔하고, 진정은 닫힌 고리라 떠난다)

> **새 가설: 사용자가 (1) 어제 신경 쓴 것의 '그 후'를 확인하고 (2) 자기만의 포모 패턴이 쌓이는 걸 보고 (3) 매일 갱신되는 '오늘의 나'를 확인하면, 재방문한다.**

3가지 차이: 신선함(흔함)이 아니라 **연속성(열린 고리)**, 집단 위로가 아니라 **개인 거울(해자)**, 진정(닫힘)이 아니라 **나의 성장 확인(계속 보고 싶은 서사)**.

### 진정의 역설 (반드시 인지)
"진정"만 팔면, 위로받은 사용자는 마음 편해져 떠난다. 위로의 완성이 곧 이탈. → 해법은 "진정"이 아니라 **"매일 달라지는 나의 흔들림 상태"**를 보게 하는 것. 어제 안 흔들렸어도 오늘 시장이 광기면 또 흔들린다. 그래서 매일 "오늘의 나는?"을 확인하러 온다.

## 4. 세 개의 화면 (스와이프 위에 얹는 레이어)

스와이프 카드 피드는 메인으로 유지. 아래 셋은 그 위/뒤/옆에 붙는다.

### 4.1 오늘의 너 (리텐션 훅) — 스와이프 끝에 뜸
카드를 다 넘기면 마지막에 뜨는 요약 한 장.
- **오늘의 흔들림 점수**(0~100) + 어제 대비 변화
- **시장 포모(집단) vs 나의 반응 대조**: "시장은 광기 84인데 너는 잘 안 흔들렸어"
- **어제→오늘 분기**(§4.4): 관심이 일관되면 "어제 그 반도체 어떻게 됐어", 튀었으면 "시선이 옮겨다니네"
- 스트릭("안 흔들린 날 5일 연속")

### 4.2 나의 포모 패턴 (해자) — 별도 탭, PRO
데이터가 쌓인 충성 사용자에게 잠금 해제(예: 2주+ 사용).
- **주간 흔들림 곡선**: 일주일을 이어 패턴을 봄. 튄 날에 사용자가 스스로 인과를 떠올림("아 그날 반도체 난리였지").
- **약한 키워드**: "너는 코인엔 단단한데 반도체엔 약해." 그 사람만의 데이터로만 만들어지는 거울 = 해자.
- **성장 지표**: "한 달 전보다 31% 덜 흔들림." 스트릭·도파민·손실회피가 여기 들어가되 방향이 반대 — **"안 한 것"을 보상**한다(안 흔들린 날 연속).

### 4.3 판단 보조 (BM 심장) — 카드 탭하면 열림
"따라가도 될까?"가 결제 트리거. 감정의 정점에서 페이월.
- **무료 = 진단**: "지금 반도체 뜨겁고, 너도 흔들리는 중이야." (불안 인지 → 훅)
- **PRO = 판단 근거 3종**(잠금):
  - **타이밍**: "이미 3일 전부터 뜨거웠어, 지금은 늦은 쪽" (객관적 기사 + 기술 지표를 쉽게 풀어서)
  - **너의 약점**: "너는 반도체에 약해, 지금이 약한 순간" (흔들림 패턴 결합)
  - **지난번의 너**: "지난달 AI도 안 따라갔잖아, 그 뒤 이렇게 됐어" (과거 데이터로 판단 근거)
- **절대 규칙**: 답("사라/마라")을 주지 않는다. 판단을 사용자에게 돌려준다. "늦은 쪽에 가까워"지 "들어가지 마"가 아니다. 면책("결정은 너의 몫")은 장식이 아니라 방패.

### 4.4 어제→오늘 분기 로직 (스키마가 안 무너지게)
**추적 단위를 "키워드"가 아니라 "사용자의 상태"로 올린다.** 관심이 매일 어디로 튀든 스키마가 항상 채워진다.
- 어제와 오늘 관심이 **일관** → "어제 그 반도체, 오늘 이렇게 됐어"(연속 추적)
- 어제와 오늘이 **튐** → "어제는 반도체, 오늘은 코인이네. 시선이 옮겨다니는 것도 포모야"(흔들림 감지)
- **핵심 통찰**: 관심이 튀는 것 자체가 포모의 증거다. 무너지는 스키마가 아니라 흔들림 점수의 입력이다.

## 5. BM (조언을 팔지 않는다는 전제 위에서)

내러티브: **무료 일일 안티-포모 피드 → 습관 + 감정적 신뢰 + 관심 데이터 → 두 자산이 복리로: (1) 관계, (2) 지수.**

| 레이어 | 무료 | PRO (구독) |
|---|---|---|
| 키워드 카드 | 오늘의 키워드 + 점수 + 친구톤 코멘트 | — |
| 용어 풀이 | 펀더멘탈·EPS를 쉽고 **따뜻하게** (불안 안 키우는 게 기준) | — |
| 오늘의 너 | 흔들림 점수 + 어제 대비 | — |
| 판단 보조 | 진단("흔들리는 중") | 판단 근거 3종(타이밍·약점·과거의 나) |
| 패턴 거울 | — | 주간 곡선 + 약한 키워드 + 성장 |

- **결제 타이밍 = 감정의 정점.** "따라가고 싶어 미치겠는데 맞는지 모르겠는" 순간에 게이트. 포모 자체가 결제 트리거. 불안 증폭이 아니라 **불안 해소의 대가**라 정체성과 안 부딪침.
- **PRO의 깊은 가치는 충성 사용자만 느낀다.** 약점·과거의 나는 데이터가 쌓여야 의미 있음 → 신규는 잠겨도 안 아깝고, 오래 쓸수록 잠긴 게 탐남 = 자연스러운 충성 게이트.
- (장기) FOMO Index 라이선싱 / (서랍) 타로 부활은 PRODUCT_TRUTH §7 참조.

## 6. 유통 전략 (SNS → 앱, 웹은 검증장 겸 랜딩)

- **장기 그림**: 매일 여는 건 앱. SNS 카드레터로 트래픽 → 앱 설치. 웹은 랜딩.
- **단, 지금은 웹 우선.** 검증 안 된 제품을 설치 마찰 뒤에 숨기면 "설치를 안 해서인지 제품이 별로인지" 구분 불가. 웹(설치 마찰 0)으로 **"매일 열 만한가"를 먼저 검증** → 증명되면 앱으로 옮기며 푸시·개인화로 리텐션 증폭.
- **시점**: 웹이 D7 재방문을 검증한 뒤 앱 착수. 그 전까지 앱(RN)은 보류(PRODUCT_TRUTH §2).
- 카드레터는 지금부터 웹으로 트래픽 보내 "다시 오는가" 측정 → 앱 출시 시 설치 전환.

## 7. Phase 1~4가 미리 깔아둘 것 (Phase 5 되돌아가기 방지)

엔진을 만들 때 아래를 염두에 둔다. 지금 구현하라는 게 아니라, **나중에 흔들림 점수가 쓸 데이터를 버리지 말라**는 뜻.

- **키워드 점수에 "언제 시작됐나"(쏠림 시작 시점) 기록**: §4.3 판단 보조의 "타이밍"이 이걸 쓴다. 일일 스냅샷에 키워드별 첫 감지일·연속일 보존.
- **사용자 행동 로그 보존**: 스와이프(관심/덜관심), 카드 탭, 본 키워드, 머문 시간. 이미 `keywordHistory`가 로컬 저장 중 → 서버 집계로 확장(PRODUCT_TRUTH §5 "개인화 신호 수집").
- **익명 세션 단위 일관성**: 흔들림 점수는 세션별 시계열이라 세션 ID가 안정적으로 유지돼야 함.
- **confidence 보존**: 흔들림 점수도 데이터 적으면 정직하게 폴백(콜드스타트, §8).

## 8. 미해결 숙제 (Phase 5 착수 전 풀 것)

다음 둘이 풀려야 §4 세 화면이 진짜가 된다. 안 풀면 또 가짜 숫자.

### 8.1 흔들림 점수 정의
**무엇으로 0~100을 만드나?** 후보 신호: 시장 광기 키워드와 내 관심의 겹침 / 스와이프 속도·양 / 평소 대비 검색·탭 증가 / 관심이 튄 정도(§4.4). 공식·가중치·정직성(가짜 점수 금지)을 별도 설계 필요.

### 8.2 콜드스타트
첫날은 비교할 어제가 없다. 첫 3일은 "너를 알아가는 중" 온보딩으로 프레이밍. 약점 분석은 2주+ 데이터 후 잠금 해제. 데이터 부족 시 점수 대신 정직한 빈 상태.

## 9. 개발 순서 & 매뉴얼

### 전체 순서
```
[지금] Phase 1 → 2 → 3 → 4 (엔진을 진짜로) — KEYWORD_ENGINE_SPEC §6
          ↓ (각 Phase 끝나면 멈추고 검증, Phase 2 후엔 실제 출력 같이 확인)
[검증] 웹에서 D7 재방문 측정 (PRODUCT_TRUTH §5 지표)
          ↓ (매일 열 만한가? 증명되면)
[Phase 5] 흔들림 점수 정의(§8.1) → 오늘의 너(§4.1) → 판단 보조 무료 진단(§4.3 무료)
          → 패턴 거울 + PRO 판단 근거(§4.2, §4.3 PRO) → 구독 결제
          ↓
[그 후] 앱(RN) 착수 + 푸시·개인화 증폭 → 카테고리 확장
```

### Phase 5 내부 순서 (엔진 검증 후)
1. **흔들림 점수 엔진**(§8.1 정의 → 순수 함수 + 테스트). 세 화면의 공통 토대라 먼저.
2. **오늘의 너**(§4.1): 점수를 스와이프 끝 카드로 노출. 어제→오늘 분기(§4.4) 포함. 무료.
3. **판단 보조 — 무료 진단**(§4.3 무료): 카드 탭 시 "흔들리는 중" 진단.
4. **패턴 거울 + PRO 게이트**(§4.2): 주간 곡선 + 약한 키워드. 2주+ 사용자 잠금 해제.
5. **PRO 판단 근거 + 결제**(§4.3 PRO): 타이밍·약점·과거의 나 + 구독.

### 운영 매뉴얼 (Phase 트랙 공통)
- **한 번에 한 Phase만.** 끝나면 Acceptance 결과 + 변경 파일 보여주고 멈춤. 다음은 직접 지시.
- **Phase별 PR로 끊어 머지.** rollback-friendly. 큰 변경 섞지 말 것.
- **민감 작업은 게이트.** prod DDL(`db push`), 결제 연동 등은 자동 실행 금지, 직접 승인.
- **정직한 숫자 원칙은 Phase 5에도 그대로.** 흔들림 점수가 임의로 나오면 즉시 폐기. confidence 노출.
- **에이전트 자율 루프는 정지 유지.** 엔진+Phase 5 안정 후 재개. 재개 전 제안 스크립트가 PRODUCT_TRUTH·PRODUCT_VISION을 읽는지 확인.
- **스와이프 UX는 건드리지 않는다.** 모든 신규 화면은 스와이프 위에 얹는 레이어.

### 검증 게이트 (Phase 5 착수 조건)
Phase 1~4 끝 + 웹에서 D7 재방문 기준선 확보 + "매일 열 만한가"에 YES. 이게 NO면 Phase 5 기능보다 엔진/콘텐츠 품질을 먼저 고친다. **기능을 더 얹기 전에 토대가 재방문을 만드는지부터 확인.**
